Техническое задание для проекта HealthTrack
1. Общие сведения
Название проекта: HealthTrack.
Тип проекта: веб‑приложение для отслеживания веса, питания и тренировок с персональной статистикой и графиками прогресса.
Автор: Кагиев Анзор.
Язык программирования: Python.
Фреймворк: Flask.

2. Цели и задачи проекта
Цель: создать веб‑приложение, позволяющее пользователям отслеживать вес, питание и тренировки, анализировать прогресс с помощью графиков и статистики.

Задачи:

реализовать регистрацию и авторизацию пользователей с безопасной аутентификацией;

предоставить функционал для ведения дневника питания и трекера тренировок;

обеспечить визуализацию прогресса в виде графиков;

гарантировать приватность данных — каждый пользователь видит только свои записи.

3. Функциональные требования
3.1. Регистрация и авторизация

создание учётной записи с указанием логина и email;

хеширование паролей с использованием bcrypt;

безопасная авторизация через Flask‑Login;

восстановление пароля (опционально).

3.2. Профиль пользователя

ввод и редактирование персональных данных: вес, рост, целевой вес;

автоматический расчёт индекса массы тела (ИМТ) по формуле: ИМТ= 
рост (м) 
2
 
вес (кг)
​
 ;

отображение текущих показателей и целевого веса.

3.3. Дневник питания

добавление записей о потреблённых продуктах с указанием:

названия продукта;

количества калорий;

даты и времени приёма пищи;

просмотр истории записей с возможностью фильтрации по дате;

подсчёт суточного потребления калорий.

3.4. Трекер тренировок

фиксация упражнений с указанием:

названия упражнения;

веса (в кг);

количества повторений;

даты тренировки;

история тренировок с возможностью редактирования и удаления записей.

3.5. Графики прогресса

визуализация динамики веса с привязкой к дате;

график суточного потребления калорий;

интерактивные графики с использованием Chart.js;

выбор периода отображения данных (неделя, месяц, год).

3.6. Приватность данных

доступ к данным только для авторизованного пользователя;

защита от несанкционированного доступа к чужим записям.

4. Технические требования
4.1. Бэкенд

язык программирования: Python 3.8+;

фреймворк: Flask;

ORM: SQLAlchemy;

аутентификация: Flask‑Login;

хеширование паролей: bcrypt.

4.2. База данных

разработка: SQLite;

продакшен: PostgreSQL или MySQL.

4.3. Фронтенд

разметка: HTML5;

стилизация: CSS3, Bootstrap 5;

графики: Chart.js;

шаблонизация: Jinja2.

6. Модели базы данных
6.1. User (учётные записи)

id (PK) — уникальный идентификатор;

username — логин пользователя;

email — электронная почта;

password_hash — хешированный пароль.

6.2. Profile (персональные данные)

id (PK);

user_id (FK) — связь с таблицей User;

weight — текущий вес (кг);

height — рост (см);

target_weight — целевой вес (кг).

6.3. FoodEntry (записи о питании)

id (PK);

user_id (FK);

food_name — название продукта;

calories — количество калорий;

date — дата записи.

6.4. Workout (записи о тренировках)

id (PK);

user_id (FK);

exercise_name — название упражнения;

weight — вес (кг);

reps — количество повторений;

date — дата тренировки.

7. Установка и запуск
7.1. Установка зависимостей

bash
pip install flask flask_sqlalchemy flask_login bcrypt
7.2. Запуск приложения

Клонируйте репозиторий.

Установите зависимости из requirements.txt.

Запустите app.py.

Откройте браузер и перейдите по адресу http://localhost:5000.

8. Критерии приёмки
все функциональные модули работают корректно;

данные пользователей защищены и доступны только им;

графики отображают актуальные данные;

интерфейс адаптивен и удобен для пользователя;

приложение стабильно работает на тестовых данных.
